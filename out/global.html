POST /login

Passport.js JWT Strategy for authenticating users. This strategy extracts the JWT token from the `Authorization` header (as a Bearer token), verifies it using the provided secret key, and retrieves the user from the database based on the decoded JWT payload.

POST /login authenticate(req, res) → {void}

POST login Authenticates the user using the local Passport strategy. If authentication is successful, a JWT token is generated and returned in the response. If authentication fails, an error message is sent back.

passport /use(username, password, callback) → {void}

Local authentication strategy using Passport.js. This strategy authenticates users by validating the `Username` and `Password` fields. The provided username is checked against the database, and the password is validated using the `validatePassword` method. If authentication is successful, the user object is returned; otherwise, an error message is passed to the callback indicating whether the username or password is incorrect.
Parameters:
Name Type Description
username string The username provided by the user for authentication.
password string The password provided by the user for authentication.
callback function The callback function to be called after authentication.
